Перенос строк.
--------------------------------------------------

Строки
не переносятся,
несмотря на то,
что переносятся. =)

А так строки  
разрываются.\
Разрываются  
строки.\
Таким образом _Enter_ в конце строки не создаёт новую строку,
a вот *двойной пробел* в конце строки создает новую строку.  
Ещё новую строку создает **обратный слэш** в конце строки \
а сам он, `of cource`, остается невидим.\
А вот если за последним обратным слэшем будет пустая строка, то он останется видим \

что и логично, ведь на него не возложена функция начала новой строки, \

она возложена на пустую строку.

# Блоки кода

**Вариант 2.** С помощью отступов.

**Вариант 3.** С подсветкой синтаксиса.

```sh
#!/bin/bash
# таким образом оформляется блок кода с подсветкой синтаксиса

case $corner in 
	45 )    echo "НЕ ПЕРЕСЕКАЕТ" ; ((misses++)) ;; 
	0 )     if [ $distance -lt $HALF_OF_WIDTH ] ; then 
			echo пересекает ; ((hits++)) ; else 
			echo "НЕ ПЕРЕСЕКАЕТ" ; ((misses++)) 
		fi ;; 
	* )     # Вычисление гипотенузы 
		hypotenuse=`echo "$distance / (c($PI/180*$corner))" | bc -l` 
		hypotenusee=`echo "scale=0; $hypotenuse / 1" | bc` 
		echo "Гипотенуза: $hypotenusee" 
		if [ $hypotenusee -lt $HALF_OF_WIDTH ] ; then 
			echo пересекает ; ((hits++)) ; else 
			echo "НЕ ПЕРЕСЕКАЕТ" ; ((misses++)) 
		fi ;; 
esac
```

Ссылки
--------------------------------------------------

где-то описано, что ссылки становятся ссылками, только будучи заключены в <>

Очевидно это *не так*.

Красивое оформление [ссылки](http://example.com/ "Необязательная подсказка")  

Ещё более [красивое][id]
>примечание: \[id] - это просто произвольные символы.   
Можно использовать даже [русские][РУ] символы, хотя не думаю, что это `good idea`  
[Цифры][1] тоже можно

В этом случае [названия][nazvanie] ссылок вставляются в одном месте, а URL и описания в другом (обычно внизу).

Таким образом удобно делать несколько, например [две][link], [три][link], [четыре][link], [десять][link], [ссылок][link] на один [URL][wiki].

[id]: http://example.com/ "Необязательная подсказка номер два "
[nazvanie]: http://yandex.ru/ "Ссылка на Яндекс"
[link]:www.mail.ru "Ссылка на мылору"
[wiki]: https://ru.wikipedia.org/wiki/URL "Вики"
[РУ]: www.gramota.ru "великий и могучий"
[1]: www.one.ru

>примечание:  
Не знаю можно ли оставлять пробелы между [..]: и URL. Вроде работает с ними.

* __Плагины для nvim__
 
1. vim-плагин [vim-instant-markdown](https://github.com/suan/vim-instant-markdown)
Работает, но мне не нравится следующее:
   - поглюкивает 
   - автопрокрутка живет своей жизнью.
   - форматирование не красивое.

по readme, но у меня так и не заработало) только после сохранения файла.

3. chrome-плагин [Markdown Preview Plus](https://chrome.google.com/webstore/detail/markdown-preview-plus/febilkbfcbhebfnokafefeacimjdckgl/related?hl=ru)
Вроде бы всё неплохо. Минуса:
   - обновление только по факту сохранения файла
   - никакой тебе автопрокрутки.
   - файл нужно открытьвать вручную
   - таблицы не компилятся
   - не фурычит vimium
